# Security Policy

Askaide takes security reports seriously and welcomes responsible disclosure.

## Reporting A Vulnerability

Please report security concerns through the official contact channel:

https://askaide.com/fr/contact

Include:

- affected URL, application surface or feature;
- clear reproduction steps;
- expected security impact;
- screenshots, request samples or logs if useful;
- your contact details for follow-up.

## Responsible Testing Rules

Please do not:

- access, modify, delete or exfiltrate user data;
- perform destructive testing;
- run denial-of-service testing;
- bypass payment systems;
- test private accounts without permission;
- publish vulnerability details before Askaide has had a reasonable opportunity to investigate.

## Public Scope

Public surfaces may include:

- `https://askaide.com`;
- public web pages;
- public documentation;
- public mobile app behavior.

Private systems, accounts, dashboards, administrative routes, infrastructure and third-party services are out of scope unless explicitly authorized.

## Response

Askaide will review valid reports and prioritize fixes based on severity, exploitability and user impact.
